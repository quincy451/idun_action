ALINK SPEC

Purpose

ALINK links ActionC64U objects into a directly launchable Commodore PRG.

Inputs

- OBJ/<MODULE>.OBJ
- dependency objects from project and library directories
- runtime helper modules selected by symbol/import reachability

Output

- BIN/<MODULE>.PRG

Responsibilities

- parse object records
- find the module entry procedure
- resolve local and external references
- load the required dependency closure
- dead-strip unreachable code/data
- select and place runtime helper modules
- build the final PRG load image
- write the final PRG through the UDOS file ABI

Object Code Records

`OBJ1` supports machine-code and relocation records:

- `m <hex bytes>`
- `r <offset> u<index>`

For the first code-section slice, ALINK accepts a root export whose body marker
is `b M`, validates the decoded `m` byte count against the export size, prefixes
the standard PRG load address, and writes those bytes directly into the final
PRG.

For the multi-module machine-code slice, a body marker ending in `M` may contain
one or more import tokens such as `b u0M` or `b u0u1M`. ALINK places the root
`m` bytes first, then places every resolved dependency object's `m` bytes in
dependency-queue order. Each `r <offset> u<index>` record patches a 16-bit
little-endian operand in the current object's `m` bytes to the placed address
of the imported symbol. Multiple relocation records and transitive object-code
dependencies are supported for this slice. Symbol-relative data relocations and
section layout beyond the single `m` code stream are later extensions.

Runtime Contract

The emitted PRG is the program presented to UDOS for launch. ALINK decides what
must be included. A linked program must not require a separate generic runtime
program to interpret or host it.

Current Default

`ALINK MAIN` writes `BIN/MAIN.PRG`.
