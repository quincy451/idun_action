ActionC64U MATH1 additions for the native C64U fork
====================================================

Source of record: lib/math1.act
Target: C64 6502 generated program; no UDOS calls or resident service required.
Numeric base: IEEE-754 binary32 REAL, round-to-nearest/ties-to-even core runtime.

Implementation status: complete in the Idun/Alpine fork. The source has 8
public constants, 43 public routines, and no private source helper. The resulting 51
public help features are compiled and linked by the Linux ACTC/ALINK path and
have generated-PRG VICE coverage. The native UDOS compiler currently exposes
FAbs, FSqrt, FSign, FTrunc, FFloor, FCeil, FRound, FFrac, FMod, FHypot, FPow, FExp,
FLn, FLog2, FLog10, FSin, FCos, FTan, FATan, FATan2, FASin, FACos, FSec, FCsc, FCot, FASec, FMin, FMax, FClamp, DegToRad, RadToDeg, PrintR, and PrintRE. Both compilers
lower FTrunc, FFloor, FCeil, FRound, FFrac, FMod, FHypot, FPow, FExp, FLn, FLog2, FLog10, FSin, FCos, FTan, FATan, FATan2, FASin, FACos, FSec, FCsc, FCot, FASec, DegToRad, and
RadToDeg to shared link-selected OBJs; the native
selected FSign/FMin/FMax/FClamp semantics also match this source. General
native REAL source lowering and the remaining 10 public routines are tracked in
docs/udos_feature_parity.md.
Range reduction remains private target runtime code.
The native `INCLUDE "MATH1"` header now also exposes all 8 constants without
allocating target storage or selecting a runtime object.

The constants and routines below are public after INCLUDE "MATH1".  The
transcendental routines use portable Action implementations or independently
selected runtime modules over the binary32 add/subtract/multiply/divide,
conversion, absolute-value, and square-root runtime. They do not use the BASIC
ROM floating-point package.

IEEE and domain behavior
------------------------

- NaN propagates through ordinary calculations and invalid domains return NaN.
- FLn(0) returns negative infinity; FLn(negative) returns NaN.
- FExp overflows to positive infinity and underflows toward positive zero.
- FASin/FACos require -1 through 1.
- FASin preserves signed zero, returns signed pi/2 at the endpoints, and
  returns canonical quiet NaN for NaN or an out-of-domain magnitude.
- FACos evaluates binary32 pi/2 minus FASin in target operation order, returns
  positive zero at +1 and binary32 pi at -1, and shares the FASin domain policy.
- FSec evaluates binary32 `1.0/FCos(radians)` in target operation order and
  inherits ordinary signed-zero, infinity, and NaN division behavior.
- FCsc evaluates binary32 `1.0/FSin(radians)` in target operation order and
  inherits ordinary signed-zero, infinity, and NaN division behavior.
- FCot evaluates binary32 `FCos(radians)/FSin(radians)` in target operation
  order and inherits ordinary signed-zero, infinity, and NaN division behavior.
- FASec evaluates binary32 `FACos(1.0/value)` in target operation order. Values
  outside its real domain inherit FACos canonical-NaN behavior.
- FACosh requires a value at least 1.
- FATanh accepts -1 through 1; endpoints return signed infinity.
- FPow accepts a negative base only when the exponent is an exact integer.
- FTrunc preserves NaN payloads, infinities, signed zero, and integral values;
  finite nonintegers truncate toward zero.
- FFloor preserves NaN payloads, infinities, signed zero, and integral values;
  finite nonintegers round toward negative infinity.
- FCeil preserves NaN payloads, infinities, signed zero, and integral values;
  finite nonintegers round toward positive infinity.
- FRound preserves NaN payloads, infinities, signed zero, and integral values;
  finite nonintegers round to the nearest integer with halfway cases away from zero.
- FFrac returns `value-FTrunc(value)`, preserving the sign of every nonzero
  finite fractional part and using the ordinary subtraction policy for exceptional values.
- FMod returns `value-FTrunc(value/divisor)*divisor`; NaN, zero divisor, and
  infinite dividend return canonical quiet NaN, while a finite dividend with
  an infinite divisor is returned bit-for-bit.
- FHypot computes a scaled square root of `x*x+y*y`, returns positive zero for
  two zero inputs, and gives infinity precedence when paired with NaN.
- Division-based functions naturally return infinity or NaN at poles according
  to the binary32 runtime.
- Core IEEE arithmetic is correctly rounded.  Transcendental results use
  range reduction plus finite polynomial/series approximations and are not
  promised to be correctly rounded in the last bit.

Porting checklist for the native C64U tree
-----------------------------------------

1. Keep the public signatures above synchronized; FTrunc, FFloor, FCeil, FRound, FFrac, FMod, FHypot, FPow, FExp, FLn, FLog2, FLog10, FSin, FCos, FTan, FATan, FATan2, FASin, FACos, FSec, FCsc, FCot, and FASec are now
   declarations backed by shared link-selected runtime objects in both products.
2. Supply the same link-selected binary32 primitives for PrintR, PrintRE,
   FAbs, FSqrt, FTrunc, FFloor, FCeil, FRound, FFrac, FMod, FHypot, FPow, FExp, FLn, FLog2, FLog10, FSin, FCos, FTan, FATan, FATan2, FASin, FACos, FSec, FCsc, FCot, FASec, add, subtract, multiply, divide, compare, INT
   conversion, and REAL conversion.
3. Keep RT_F_WRAP_PI private target runtime code rather than a public MATH1
   source routine.
4. Add the constants and all public signatures to ACTEDIT/ACTHELP library help.
5. Run known-value target tests for positive and negative FATan, all FATan2
   quadrants, FASin/FACos endpoints/domain cases, FSin/FCos/FTan/FSec/FCsc/FCot/FASec, logs, powers, and
   hyperbolic functions.  The Idun
   fork's VICE regression executes twenty-one representative results on a 6502.

Small example
-------------

INCLUDE "MATH1"

MODULE MAIN

PROC MAIN()
    REAL angle,x,y
    angle=DegToRad(45.0)
    x=FCos(angle)
    y=FSin(angle)
    PrintR(FHypot(x,y))
    RETURN
ENDPROC
