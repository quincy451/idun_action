ActionC64U MATH1 additions for the native C64U fork
====================================================

Source of record: lib/math1.act
Target: C64 6502 generated program; no UDOS calls or resident service required.
Numeric base: IEEE-754 binary32 REAL, round-to-nearest/ties-to-even core runtime.

Implementation status: complete in the Idun/Alpine fork. The source has 8
public constants, 43 public routines, and 1 private helper. The resulting 51
public help features are compiled and linked by the Linux ACTC/ALINK path and
have generated-PRG VICE coverage. The native UDOS compiler currently exposes
FAbs, FSqrt, FSign, FMin, FMax, PrintR, and PrintRE. Its dependency-free FSign
helper and FMin/FMax selector semantics match this source, while general REAL
source lowering and the remaining library work are tracked in
docs/udos_feature_parity.md.

The constants and routines below are public after INCLUDE "MATH1".  The
transcendental routines are portable Action implementations over the binary32
add/subtract/multiply/divide, conversion, absolute-value, and square-root
runtime.  They do not use the BASIC ROM floating-point package.

IEEE and domain behavior
------------------------

- NaN propagates through ordinary calculations and invalid domains return NaN.
- FLn(0) returns negative infinity; FLn(negative) returns NaN.
- FExp overflows to positive infinity and underflows toward positive zero.
- FASin/FACos require -1 through 1.
- FACosh requires a value at least 1.
- FATanh accepts -1 through 1; endpoints return signed infinity.
- FPow accepts a negative base only when the exponent is an exact integer.
- Division-based functions naturally return infinity or NaN at poles according
  to the binary32 runtime.
- Core IEEE arithmetic is correctly rounded.  Transcendental results use
  range reduction plus finite polynomial/series approximations and are not
  promised to be correctly rounded in the last bit.

Porting checklist for the native C64U tree
-----------------------------------------

1. Port lib/math1.act as source, retaining the public signatures above.
2. Supply the same link-selected binary32 primitives for PrintR, PrintRE,
   FAbs, FSqrt, add, subtract, multiply, divide, compare, INT conversion, and
   REAL conversion.
3. Keep _MathWrapPi private; help/catalog builders intentionally ignore names
   beginning with an underscore.
4. Add the constants and all public signatures to ACTEDIT/ACTHELP library help.
5. Run known-value target tests for positive and negative FATan, all FATan2
   quadrants, FSin/FCos/FTan, logs, powers, and hyperbolic functions.  The Idun
   fork's VICE regression executes sixteen representative results on a 6502.

Small example
-------------

INCLUDE "MATH1"

MODULE MAIN

PROC MAIN()
    REAL angle,x,y
    angle=DegToRad(45.0)
    x=FCos(angle)
    y=FSin(angle)
    PrintR(FHypot(x,y))
    RETURN
ENDPROC
